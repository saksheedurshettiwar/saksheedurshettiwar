Gloria Dorsey · Hospital · Org Portal
Post a Job Requisition with Compliance Pack
Gloria needs to post an ICU nurse opening — select the right template, attach the compliance pack, set shift dates, and submit. Once she hits submit, requirements lock permanently via the Snapshot Rule.
Entry: Org Portal Dashboard
Success: Job live · Requirements locked
~4 min avg task time
🎉
Job is live. Compliance requirements permanently locked. All vendors notified instantly.
Gloria sees the job in Active status. Snapshot Rule applied — nobody can change what was required retroactively.
Edge cases No compliance pack — blocked from submitting End date before start date — inline validation error No vendors for specialty — warning, can still post Draft auto-saved if browser closes mid-flow
Marcus Webb · Staffing Agency · Vendor Portal
Submit a Candidate + Close Compliance Gaps
Marcus sees a new ICU job. He checks his roster against the requirements, selects Sarah, submits her, then uploads the 2 missing documents so admin can verify and the placement can activate.
Entry: Vendor Portal · Open Jobs
Success: Candidate submitted · Gaps uploaded · Admin notified
~8 min avg task time
📋
Candidate submitted. Gap docs uploaded. Admin verification task created. Placement locked pending review.
Marcus sees real-time upload status. Gloria sees "Locked — 2 gaps". No phone call needed. No email chain.
Edge cases Wrong file format — inline format error Expired document uploaded — system flags immediately Hospital rejects candidate — Marcus notified, can resubmit another Job filled mid-flow — job marked closed, Marcus warned
Sarah M. · Travel Nurse · Candidate Portal
Upload a Document to Wallet + Track Gap Status
Sarah receives a gap notification — 2 items missing for her St. Mary's placement. She uploads her State RN License directly, flags the Background Check to Marcus, and tracks verification status in real time until the placement unlocks.
Entry: Gap notification · Candidate Portal
Success: Docs uploaded · Status tracking live · Placement unlocks
~5 min avg task time
👛
State RN License uploaded and auto-tagged. Background Check flagged to Marcus. Sarah tracks both in real time.
When admin approves both, placement activates automatically. Sarah gets confirmation before Day 1 — not at the reception desk.
Edge cases Uploads an expired document — expiry warning shown immediately Wrong file type — inline error with accepted formats Admin rejects document — Sarah notified with reason Marcus doesn't upload BG Check in 72hrs — auto-escalation alert
Platform Admin · NexusForce Operations · Admin Portal
Verify Documents + Activate Placement
Admin receives a verification task for Sarah's St. Mary's placement. They open the placement file, read both uploaded documents carefully, approve or reject each one, and trigger activation. The compliance snapshot locks permanently the moment they approve — creating an immutable, legally defensible record.
Entry: Admin Portal · Verification queue
Success: Docs approved · Placement activated · Snapshot locked
~6 min avg task time
🔒
Both documents verified. Placement activated. Compliance snapshot locked permanently. All 3 portals updated simultaneously.
Immutability Rule fires: what was required, submitted, and verified is now frozen. Timestamped. Legally defensible. Nobody can edit this record.
Edge cases Document is expired — admin rejects · vendor must re-upload Document appears fraudulent — admin flags · placement frozen · ops alerted Partial approval (one valid, one not) — partial status shown to all Task unreviewed 48hrs — escalation reminder to admin Cannot activate placement with any open gap — system blocks it
